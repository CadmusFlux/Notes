    CSS is the language we use to style an HTML document.

    CSS describes how HTML elements should be displayed.

    Types of CSS:

    • Inline CSS: Inline CSS contains the CSS property in the body section attached with element is known as inline CSS. This kind of style is specified within an HTML tag using the style attribute.
    • Internal CSS: This can be used when a single HTML document must be styled uniquely.
    • External CSS: External CSS contains separate CSS file which contains only style property with the help of tag attributes (For example class, id, heading, … etc). CSS property written in a separate file with .css extension and should be linked to the HTML document using link tag. This means that for each element, style can be set only once and that will be applied across web pages.

  • CDATA v/s PCDATA

    CDATA stands for unparsed character data. The tags inside a CDATA tag are not considered as markup and are a whole entity,

    PCDATA stands for Parsed character data . The tags inside PCDATA will be considered as markup and parsed by XML parsers.

    CDATA: 

	<employee>  
	<![CDATA[  
	  <firstname>vimal</firstname> 
	  <lastname>jaiswal</lastname> 
	  <email>vimal@javatpoint.com</email> 
	]]>   
	</employee>

output : <firstname>vimal</firstname> <lastname>jaiswal</lastname> <email>vimal@javatpoint.com</email>

PCDATA:
 
	<employee>  
	<![CDATA[  
	  <firstname>vimal</firstname> 
	  <lastname>jaiswal</lastname> 
	  <email>vimal@javatpoint.com</email> 
	]]>   
	</employee>

Output : Vimal Jaiswal vimal@dasdad.com

    Keys must be strings, and values must be a valid JSON data type (string, number, object, array, boolean or null)

  • JSON v/s XML
    • JSON object has a type whereas XML data is typeless.
    • JSON does not provide namespace support while XML provides namespaces support.
    • JSON has no display capabilities whereas XML offers the capability to display data.
    • JSON is less secured whereas XML is more secure compared to JSON.
    • JSON supports only UTF-8 encoding whereas XML supports various encoding formats.

  • Struts

    Struts is a web-application framework used to create a web applications based on servlet and JSP. Struts depend on the MVC (Model View Controller) framework.

    Features of Struts:

    • Struts is almost simple, so easy to learn and use.
    • Struts encourages good design practices and modeling because the framework is designed with “time-proven” design patterns.
    • input validation and internationalization.
    • It also hand over standard error handling both programmatically and declaratively.

    Components of Struts

    1. JSP Programs (View Layer Resources)
    1. FormBean Class(Java Class) (Controller Layer Resources)
    1. Action Servlet(Built-in Controller Servlet) (Controller Layer Resources)
    1. Action Class (Java Class) (Controller Layer Resources)
    1. web.xml (Deployment Descriptor file of web application)
    1. Struts Configuration File (XML File) (Controller Layer Resources)
