  • Hubs v/s Switch:
    • Hubs
      • Hubs are common connection devices that connect all devices to one point.
      • The data transmitted by 1 pc will be transmitted to all the device on the HUB .
      • A hub is a box that takes a signal in one port and sends it out all the other ports.
      • E.g : Walkie-Talkie.
    • Switch:
      • A switch is a box that takes in a signal in on one port and sends it out to only relevant ports based on MAC address.
      • Switch is similar to Hub just it is more intelligent in the sense that it can transmit to specific devices.
      • These devices are each given ports. So when a packet arrives for a specific devices it is routed to the port corresponding to that device.
      • MAC addresses of pc are used to give ports.

  • IP address - Internet protocol address

    Every device that is connected to the internet and uses Internet protocol for communication has an IP address.

    IP address 192.168.0.1 → each octet ranges from 0 to 255.

  • IP classes

    Every IP address has 2 parts : first determines the network it is on and the second determines the devices on that network.

    The length of the first part changes based on IP classes:

    IP address: A.B.C.D (OR) 192.168.0.1

    Class A (1-127) : Everything before the first period indicates the network, and everything after it specifies the device within that network

    Class B (128-191): Everything before the second period indicates the network. .

    Class C (192-223.): For Class C networks, everything before the third period indicates the network.

    Class D (224-239)

    Class E (240-254)

  • OSI - Open Systems Interconnection

    It is a conceptual reference for how computers communicate over network.

    There are seven layers:

    1. Physical layer - Electrical signals to bits [ HUB, 100BaseTX, wires , cables ]
    1. Data Link layer (frames) - [SWITCH, PPP, Frame Relay, ATM, Fiber Cable, etc.]
    1. Network Layer (packets)- To move packets from source to destination. [Router/IP]
    1. Transport Layer (segments)- applies protocol on network layer to trnsfr data . [ TCP/UDP]
    1. Session - To establish, manage, and terminate the session. [NetBIOS, SAP]
    1. presentation - To translate, encrypt and compress data. [MPEG, ASCH, SSL, TLS]
    1. Application - To allow access to network resources. [SMTP, HTTP, FTP, POP3, SNMP]

  • Switch v/s Bridge

    no.SwitchBridge
    1A switch can have a lot of ports.A bridge can have 2 or 4 ports only.
    2It is a device which is responsible for channeling the data that is coming into the various input ports to a particular output port which will further take the data to the desired destination.It is basically a device which is responsible for dividing a single network into various network segments.
    3The switch performs the packet forwarding by using hardwares such as ASICS hence, it is hardware based.The bridge performs the packet forwarding by using softwares so it is software based.
    4The switching method in case of a switch can thus be store, forward, fragment free or cut through.The switching method in case of a bridge is store and forward.
    5The task of error checking is performed by a switch.A bridge cannot perform the error checking.

  • Topology

    Topology defines the structure of the network

    • Mesh Topology : every device is connected to another device via the particular channel
    • Star Topology : In star topology, all the devices are connected to a single hub through a cable.
    • Bus Topology :Bus topology is a network type in which every computer and network device is connected to a single cable.
    • Ring Topology: Every device has exactly two neighboring devices for communication purpose
    • Peer to Peer (P2P) : The network consists of a direct link between two computers
