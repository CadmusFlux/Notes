  • Deadlock

    In an operating system, a deadlock occurs when a process or thread enters a waiting state because a requested system resource is held by another waiting process, which in turn is waiting for another resource held by another waiting process.

    If a process is unable to change its state indefinitely because the resources requested by it are being used by another waiting process, then the system is said to be in a deadlock.

    Deadlock is a situation where a set of processes are blocked because each process is holding a resource and waiting for another resource acquired by some other process.

  • Deadlock Prevention

    Deadlock prevention works by preventing one of the four Coffman conditions from occurring.

    1. Remove Mutual Exclusion:

      One or more than one resource are non-shareable (Only one process can use at a time)

      Remove this by making resources available to all process and exclusion to one is removed.

    1. Remove Hold and Wait:

      A process is holding at least one resource and waiting for resources.

      Removing this by making the program define which resources it need before execution so that it will be executed only when everything is available.

    1. No Preemption:

      A resource cannot be taken from a process unless the process releases the resource.

    1. Circular Wait:

      A set of processes are waiting for each other in circular form.

  • Deadlock Detection

    Deadlock detection is the process of actually determining that a deadlock exists and identifying the processes and resources involved in the deadlock

  • Pass : Each read of the src code is called pass.

    The assembler reads the assembly language source code twice before it outputs object code. Each read of the source code is called a pass.

    Pass I :

    1. Define symbols and literals and store them in symbol tables.
    1. Keep track of location counter.
    1. Process Assembly directives or psuedo-opcode.

    Pass II :

    1. Generate machine code by converting mnemonic code to numeric code.
    1. Generate data for literals and look for values of symbols

  • Page Fault

    If the referred page is not present in the main memory then there will be a miss and the concept is called Page miss or page fault.

    Explanation:

    The referred page points to an actual memory location and if that actual memory location is not present than it will raise a page fault.

  • Page replacement

    In operating systems, whenever a new page is referred and not present in memory, page fault occurs and Operating System replaces one of the existing pages with newly needed page. Different page replacement algorithms suggest different ways to decide which page to replace. The target for all algorithms is to reduce number of page faults.

    1. LRU (Least recently used page replacement algo) is greedy algorithm for page replacement where the page to be replaced is least recently used.
    1. Optimal page replacement algorithm :- OS replaces the page that will not be used for the longest period of time in future
  • Preemptive and Non preemptive

    Preemptive scheduling is process scheduling technique in which CPU time is divided into slots and each process is provided with a certain slot of time only. If the process doesn't complete in the provided timeframe it is sent to the ready queue and waits for the next chance. E.g :- Round-robin , priority ,SRTF

    Non-preemptive scheduling is a scheduling technique in which a process takes the CPU resource and holds it until the process is completely done. A process mid execution cannot be stopped. E.g : non-pre-emptive , priority and SJF

  • Physical Memory v/s Virtual Memory

    1) Physical memory :

    Physical memory is the actual memory i.e ( RAM) connected to the processor .

    It is in the from of sticks that are connected to the mother-board.

    2) Virtual memory :

    Virtual memory is a memory management technique which provides abstraction or mapping of the actual memory present in the system.

    A computer can address more memory than the amount physically installed on the system. This extra memory is actually called virtual memory and it is a section of a hard disk that's set up to emulate the computer's RAMThe ARM® assembler reads the assembly language source code twice before it outputs object code. Each read of the source code is called a pass.

    Virtual memory helps in hiding the fragmented storage system in the main memory.

    Explanation :

    Virtual memory are divided into logical spaces called pages. this is analogous to physical memory divided into spaces called addresses.

    The pages are stored in a table called page table , which is a two-column mapping of pages to actual addresses.

    Now , programs refer to these pages rather than actual addresses. The page points to the actual address.

    The advantage this provides is hiding the actual memory size and the memory contents from programs.

    Also it helps in making the memory look-like contagious memory rather than fragmented one by using consecutive pages . These can than refer to non-continous addresses but it will trick the program into thinking it is contiguous.

  • Scheduling : assigns tasks to hardware
    • Scheduling is the method by which tasks are assigned and managed between hardware resources (such as CPU , peripheral-devices , memory devices, network devices) .
    • Types/algorithms of scheduling : 6
      1. First Come First Serve : FIFO simply queues processes in the order they arrived from the ready queue.
      1. Shortest Remaining Time : The processes which has lowest time until completion is executed first .
      1. Priority Based Scheduling : Each task is assigned a deadline or priority . Based on this priority tasks are executed.
      1. Round - Robin : The scheduler assigns the resources to each process for a unit time and then moves on to the next process . It does so in a cycle until processes are completed.
      1. Shortest Job First : The job/process with shortest execution time is executed first.
      1. Multi-Level queues : Ready queue is divided into different queues based on property such as priority , memory size , etc.
    • Scheduler : It is a OS module that selects the next jobs to be performed and processes to run. Scheduler is what carries out the scheduling.
    • Types of scheduler : 3
      1. Process scheduler : Decides which processes are to be executed.
      1. Long term scheduler : Decides which jobs/process should be admitted to ready queue ( ready for execution)
      1. Medium/mid term scheduler : Swaps inactive processes out of main memory and into second memory and vice-versa.
      1. Short-term scheduler : Executes processes present in ready queue
      1. Dispatcher : Dispatcher is a component of scheduler which gives access of the CPU to the process selected by the short-term scheduler.

  • Scheduling Criteria :

    Maximize :

    1. CPU Utilization : Ensuring CPU is busy as much as possible.
    1. Throughput : Jobs/Processes completed per unit time.

    Minimize:

    1. Waiting Time : The time that a process has to wait in a queue
    1. Response time : The time interval between request and response between process and scheduler/CPU
    1. Turn-around time : The time interval between dispatching resources to a process until it's completion.

  • Thrashing :

    Thrashing occurs when computer's virtual memory are overuses which leads to constant state of paging and page faults. This slows down application processes.

    This can continue indefinitely if resource extensive programs are not closed or virtual memory is freed up.
